বাংলায় Theoretical Summary বাংলা

🔷 RNN কী? কেন দরকার?

Standard Neural Network-এ প্রতিটি input independent। কিন্তু sequence data-তে (text, speech, time series) আগের information পরের decision-এ কাজে লাগে। RNN এই কাজটা করে — প্রতিটি time step-এ আগের hidden state hₜ₋₁ নিয়ে নতুন hₜ তৈরি করে।

সূত্র: hₜ = tanh(U·xₜ + W·hₜ₋₁) এবং output: ŷₜ = softmax(V·hₜ)

  • U = input weight matrix, W = recurrent weight matrix, V = output weight matrix
  • প্রতিটি time step-এ same weights (U, W, V) ব্যবহার করা হয়

🔷 BPTT (Backpropagation Through Time)

RNN-এ loss হিসাব করতে time-এর উপর দিয়ে backprop করতে হয়। Total gradient:

∂E/∂W = Σₜ ∂Eₜ/∂W — প্রতিটি time step-এর gradient যোগ হয়।

t=3 এ loss-এর জন্য:

∂E₃/∂W = Σₖ₌₀³ (∂E₃/∂ŷ₃)(∂ŷ₃/∂s₃)(∂s₃/∂sₖ)(∂sₖ/∂W)

মানে s₃ → s₂ → s₁ → s₀ পর্যন্ত chain rule apply করতে হয়। এই কারণে h গুলো memory-তে save করতে হয় — memory constraint আসে।

🔷 Vanishing Gradient Problem

BPTT-তে ∂sⱼ/∂sⱼ₋₁ একটি Jacobian matrix — এর 2-norm এর upper bound হলো 1। tanh function সব value -1 থেকে +1 এ map করে, derivative সর্বোচ্চ 1।

Chain rule-এ অনেকগুলো এই small value গুণ হলে: gradient → 0 (vanishing) অথবা বড় value থাকলে gradient → ∞ (exploding)।

ফলে, দূরের time step-এর information শেখা যায় না। "I grew up in India… I speak fluent Hindi" — এখানে "India" আর "Hindi" এর মাঝে অনেক time step থাকলে RNN ভুলে যায়।

🔷 LSTM কীভাবে এই সমস্যা সমাধান করে?

LSTM দুটো জিনিস রাখে:

  • Cell State (Cₜ) → Long-term memory — এটি একটি "highway" যেখানে information প্রায় unchanged থেকে flow করে। Gradient এখানে multiply হয় না, বরং add হয় → vanishing gradient নেই!
  • Hidden State (hₜ) → Short-term memory — বর্তমান output।

৩টি Gate (percentage calculator) → sigmoid ব্যবহার করে (0 to 1):

  • Forget Gate fₜ → আগের long-term memory কতটুকু রাখবো?
  • Input Gate iₜ + C̃ₜ → নতুন কী information যোগ করবো?
  • Output Gate oₜ → cell state-এর কতটুকু output হবে?

Constant Error Carousel: Cell state-এ weight fixed at 1 — gradient backprop করার সময় এই highway দিয়ে constant magnitude-এ যায়, vanish হয় না।

🔷 GRU — LSTM-এর simpler version

GRU-তে শুধু একটি state (hidden state hₜ) এবং মাত্র ২টি gate:

  • Update Gate zₜ → LSTM-এর forget + input gate combined। কতটুকু past info রাখবো?
  • Reset Gate rₜ → কতটুকু past info ভুলে যাবো?

GRU fewer parameters → faster training। LSTM more expressive → complex tasks-এ better।

LSTM — Full Algorithm & End-to-End Example Math

Setup: Input dimension = 2, Hidden dimension = 2 (scalar for simplicity)
Sequence: 2 time steps. x₁ = [1, 0], x₂ = [0, 1]
Initial state: h₀ = [0, 0], C₀ = [0, 0]
All weights simplified to scalars for clarity. sigmoid(x) = 1/(1+e⁻ˣ), tanh values given.
The 4 LSTM Equations (Memorize These!)
━━━━━━━ LSTM CORE EQUATIONS ━━━━━━━
FORGET GATE: fₜ = σ(Wf·[hₜ₋₁, xₜ] + bf) ← how much of OLD memory to keep? (0=forget all, 1=keep all)
INPUT GATE: iₜ = σ(Wi·[hₜ₋₁, xₜ] + bi) ← how much of NEW info to write?
CANDIDATE: C̃ₜ = tanh(Wc·[hₜ₋₁, xₜ] + bc) ← what is the new candidate info? (range -1 to +1)
OUTPUT GATE: oₜ = σ(Wo·[hₜ₋₁, xₜ] + bo) ← how much of cell state to expose?

━━━ STATE UPDATES ━━━
Cₜ = fₜ ⊙ Cₜ₋₁ + iₜ ⊙ C̃ₜ ← new long-term memory (⊙ = element-wise multiply)
hₜ = oₜ ⊙ tanh(Cₜ) ← new short-term memory / output
Step 1: Forward Pass at t=1 (x₁ = [1, 0])
Assumed weights (simplified for calculation):
Wf = Wi = Wc = Wo = 0.5 (for input connection)
Uf = Ui = Uc = Uo = 0.3 (for hidden state connection)
All biases = 0.  h₀ = 0, C₀ = 0, x₁ = 1 (scalar, using first element for clarity)
━━━━━━━ TIME STEP t=1, Input x₁=1, h₀=0, C₀=0 ━━━━━━━

# Raw linear combination for all gates: W·xₜ + U·hₜ₋₁ + b
linear = Wf·x₁ + Uf·h₀ + bf = 0.5×1 + 0.3×0 + 0 = 0.50

━━ FORGET GATE ━━
f₁ = σ(0.50) = 1/(1 + e⁻⁰·⁵) = 1/(1 + 0.6065) = 0.622
Meaning: keep 62.2% of old memory (but C₀=0 so no effect here)

━━ INPUT GATE ━━
i₁ = σ(0.50) = 0.622 ← how much new info to write

━━ CANDIDATE ━━
C̃₁ = tanh(0.50) = 0.462 ← what new info to potentially store

━━ OUTPUT GATE ━━
o₁ = σ(0.50) = 0.622 ← how much cell state to expose

━━━ STATE UPDATES ━━━
C₁ = f₁ × C₀ + i₁ × C̃₁
= 0.622 × 0 + 0.622 × 0.462
= 0 + 0.287
= 0.287 ← new long-term memory

h₁ = o₁ × tanh(C₁)
= 0.622 × tanh(0.287)
= 0.622 × 0.281
= 0.175 ← new hidden state / output
Step 2: Forward Pass at t=2 (x₂ = 0, h₁ = 0.175, C₁ = 0.287)
━━━━━━━ TIME STEP t=2, Input x₂=0, h₁=0.175, C₁=0.287 ━━━━━━━

linear = W·x₂ + U·h₁ + b = 0.5×0 + 0.3×0.175 + 0 = 0 + 0.053 = 0.053

━━ FORGET GATE ━━
f₂ = σ(0.053) = 1/(1+e⁻⁰·⁰⁵³) = 1/(1+0.948) = 0.513
Meaning: keep only 51.3% of C₁ — model is partially forgetting

━━ INPUT GATE ━━
i₂ = σ(0.053) = 0.513

━━ CANDIDATE ━━
C̃₂ = tanh(0.053) = 0.053 (tanh≈x for small x)

━━ OUTPUT GATE ━━
o₂ = σ(0.053) = 0.513

━━━ STATE UPDATES ━━━
C₂ = f₂ × C₁ + i₂ × C̃₂
= 0.513 × 0.287 + 0.513 × 0.053
= 0.147 + 0.027
= 0.174 ← cell state partially carried forward

h₂ = o₂ × tanh(C₂)
= 0.513 × tanh(0.174)
= 0.513 × 0.173
= 0.089 ← final hidden state output

Key Observation: At t=2 the input is 0, but hₜ and Cₜ are non-zero — the LSTM remembered information from t=1 via the cell state. A standard RNN with just tanh would have this memory squeezed and potentially vanish. The cell state Cₜ persisted because forget gate ~0.51 kept it alive.
Why LSTM Doesn't Have Vanishing Gradient
Standard RNN gradient chain:
∂sₜ/∂s₀ = ∏ⱼ₌₁ᵗ (∂sⱼ/∂sⱼ₋₁) = ∏ⱼ₌₁ᵗ tanh'(·) × W ← each term < 1, product → 0

LSTM cell state gradient chain:
∂Cₜ/∂Cₜ₋₁ = fₜ ← just the forget gate value (learned, not squeezed by tanh)

∂Cₜ/∂C₀ = ∏ⱼ₌₁ᵗ fⱼ ← product of forget gates ≈ 1 if network learns to keep them near 1

This is the "Constant Error Carousel" — gradient flows through Cₜ unchanged,
no squashing function in the direct path → vanishing gradient solved!

GRU — Full Algorithm & End-to-End Example Math

The 4 GRU Equations (Memorize These!)
━━━━━━━ GRU CORE EQUATIONS ━━━━━━━
UPDATE GATE: zₜ = σ(Wz·xₜ + Uz·hₜ₋₁) ← how much past info to carry forward?
RESET GATE: rₜ = σ(Wr·xₜ + Ur·hₜ₋₁) ← how much past info to FORGET when computing new info?
CANDIDATE: h̃ₜ = tanh(Wh·xₜ + rₜ ⊙ Uh·hₜ₋₁) ← new candidate hidden state (reset gate applied)
NEW HIDDEN: hₜ = (1−zₜ) ⊙ hₜ₋₁ + zₜ ⊙ h̃ₜ ← interpolate: keep old OR use new

━━━ Intuition of the final equation ━━━
If zₜ=0: hₜ = 1×hₜ₋₁ + 0×h̃ₜ = hₜ₋₁ ← completely ignore new input, keep old hidden state
If zₜ=1: hₜ = 0×hₜ₋₁ + 1×h̃ₜ = h̃ₜ ← completely replace with new candidate
If zₜ=0.3: hₜ = 0.7×hₜ₋₁ + 0.3×h̃ₜ ← mix: 70% old + 30% new
Same setup as LSTM: xₜ scalar, hₜ scalar.
Wz = Wr = Wh = 0.5, Uz = Ur = Uh = 0.3, all biases = 0.
h₀ = 0, Sequence: x₁ = 1, x₂ = 0
Step 1: GRU Forward at t=1 (x₁=1, h₀=0)
━━━━━━━ TIME STEP t=1, x₁=1, h₀=0 ━━━━━━━

linear = Wz·x₁ + Uz·h₀ = 0.5×1 + 0.3×0 = 0.50

━━ UPDATE GATE ━━
z₁ = σ(0.50) = 0.622 ← 62.2% new info, 37.8% old

━━ RESET GATE ━━
r₁ = σ(0.50) = 0.622 ← keep 62.2% of hₜ₋₁ when computing candidate

━━ CANDIDATE h̃₁ ━━
h̃₁ = tanh(Wh·x₁ + r₁ ⊙ Uh·h₀)
= tanh(0.5×1 + 0.622×0.3×0)
= tanh(0.5 + 0)
= tanh(0.50) = 0.462

━━ NEW HIDDEN STATE ━━
h₁ = (1 − z₁) × h₀ + z₁ × h̃₁
= (1 − 0.622) × 0 + 0.622 × 0.462
= 0.378 × 0 + 0.622 × 0.462
= 0 + 0.287
= 0.287
Step 2: GRU Forward at t=2 (x₂=0, h₁=0.287)
━━━━━━━ TIME STEP t=2, x₂=0, h₁=0.287 ━━━━━━━

linear = Wz·x₂ + Uz·h₁ = 0.5×0 + 0.3×0.287 = 0 + 0.086 = 0.086

━━ UPDATE GATE ━━
z₂ = σ(0.086) = 1/(1+e⁻⁰·⁰⁸⁶) = 1/(1+0.917) = 0.521
← 52.1% new info, 47.9% old

━━ RESET GATE ━━
r₂ = σ(0.086) = 0.521

━━ CANDIDATE h̃₂ ━━
h̃₂ = tanh(Wh·x₂ + r₂ ⊙ Uh·h₁)
= tanh(0.5×0 + 0.521×0.3×0.287)
= tanh(0 + 0.044855)
= tanh(0.0449) = 0.0448

━━ NEW HIDDEN STATE ━━
h₂ = (1 − z₂) × h₁ + z₂ × h̃₂
= (1 − 0.521) × 0.287 + 0.521 × 0.0448
= 0.479 × 0.287 + 0.521 × 0.0448
= 0.1375 + 0.0233
= 0.161

LSTM vs GRU Side-by-Side Comparison

LSTM

  • 2 states: Cₜ (long-term) + hₜ (short-term)
  • 3 gates + 1 candidate = 4 equations
  • Separate forget & input gates
  • More parameters → slower training
  • Better for very long sequences
  • Cell state = direct gradient highway

GRU

  • 1 state: hₜ (combined memory)
  • 2 gates + 1 candidate = 3 equations (+ final hₜ)
  • Update gate = forget + input combined
  • Fewer parameters → faster training
  • Often comparable performance to LSTM
  • Simpler architecture, easier to tune

BiRNN (Bidirectional RNN) — Why & How Math

🔷 কেন BiRNN দরকার?

Standard RNN শুধু left-to-right পড়ে — আগের context দেখে পরের word predict করে। কিন্তু অনেক NLP task-এ future context-ও দরকার।

উদাহরণ: "He said, 'Teddy _____ is cute.'" — blank কী হবে? "bear" নাকি "Roosevelt"? এটা বুঝতে পরের word দেখা দরকার।

আরেকটি উদাহরণ: Named Entity Recognition-এ "Paris" → city নাকি person name? পরের context দেখলে বোঝা যায়।

BiRNN solution: দুটো RNN parallel-এ চালাও — একটা left→right, একটা right→left। তাদের hidden state concatenate করে পুরো context পাও।

BiRNN Architecture & Forward Flow
━━━━━━━ BiRNN STRUCTURE ━━━━━━━

Forward RNN (left to right):
h⃗ₜ = tanh(W⃗·xₜ + U⃗·h⃗ₜ₋₁ + b⃗) ← processes x₁, x₂, x₃, ..., xₜ in order

Backward RNN (right to left):
h⃐ₜ = tanh(W⃐·xₜ + U⃐·h⃐ₜ₊₁ + b⃐) ← processes xₜ, xₜ₋₁, ..., x₁ in reverse

Final output (concatenation):
yₜ = V·[h⃗ₜ ; h⃐ₜ] + c ← [;] means concatenate. Output sees BOTH past and future.

If hidden size = d, then [h⃗ₜ ; h⃐ₜ] has size 2d
Complete Mathematical Example — Sentiment Analysis
Task: Classify "I love this" as positive/negative.
Sequence: x₁="I", x₂="love", x₃="this" (each encoded as a number for simplicity)
x₁=0.1, x₂=0.9, x₃=0.2 | W⃗=W⃐=0.6, U⃗=U⃐=0.4, all biases=0, h₀=h₄=0
━━━━━━━ FORWARD RNN (left → right) ━━━━━━━

h⃗₁ = tanh(W⃗·x₁ + U⃗·h⃗₀) = tanh(0.6×0.1 + 0.4×0) = tanh(0.06) = 0.060
h⃗₁ captures: context from "I" only

h⃗₂ = tanh(W⃗·x₂ + U⃗·h⃗₁) = tanh(0.6×0.9 + 0.4×0.060) = tanh(0.54+0.024) = tanh(0.564) = 0.511
h⃗₂ captures: context from "I love"

h⃗₃ = tanh(W⃗·x₃ + U⃗·h⃗₂) = tanh(0.6×0.2 + 0.4×0.511) = tanh(0.12+0.204) = tanh(0.324) = 0.314
h⃗₃ captures: full left context "I love this"
━━━━━━━ BACKWARD RNN (right → left) ━━━━━━━

h⃐₃ = tanh(W⃐·x₃ + U⃐·h⃐₄) = tanh(0.6×0.2 + 0.4×0) = tanh(0.12) = 0.120
h⃐₃ captures: context from "this" only (no future)

h⃐₂ = tanh(W⃐·x₂ + U⃐·h⃐₃) = tanh(0.6×0.9 + 0.4×0.120) = tanh(0.54+0.048) = tanh(0.588) = 0.528
h⃐₂ captures: context from "love this"

h⃐₁ = tanh(W⃐·x₁ + U⃐·h⃐₂) = tanh(0.6×0.1 + 0.4×0.528) = tanh(0.06+0.211) = tanh(0.271) = 0.266
h⃐₁ captures: full right context at position 1 ("I" + saw future "love this")
━━━━━━━ COMBINED REPRESENTATIONS ━━━━━━━

Position 1 ("I"): y₁ = [h⃗₁; h⃐₁] = [0.060, 0.266]
Forward: only "I" | Backward: "I love this" future context ✓

Position 2 ("love"): y₂ = [h⃗₂; h⃐₂] = [0.511, 0.528]
Forward: "I love" | Backward: "love this" ← BOTH sides! ✓

Position 3 ("this"): y₃ = [h⃗₃; h⃐₃] = [0.314, 0.120]
Forward: full sentence | Backward: only "this"

If using last states for sequence classification:
final = [h⃗₃; h⃐₁] = [0.314, 0.266] ← forward last + backward first = full sentence representation

🔷 BiRNN কখন use করবো?

  • YES — Named Entity Recognition: "Apple is a company" vs "I ate an apple" → context দুই দিক থেকে দেখা দরকার
  • YES — Machine Translation (encoder): পুরো source sentence encode করতে BiRNN ভালো
  • YES — Sentiment Analysis: পুরো sentence-এর context একসাথে দরকার
  • NO — Language Generation: Future token দেখা যাবে না (cheating হবে), তাই unidirectional use করতে হবে
  • NO — Real-time speech recognition: Future data এখনও আসেনি

Professor's Explicit Questions & Answers English

Q1. What is the Vanishing Gradient Problem in RNN, and why does it occur?

In BPTT, the gradient of the loss with respect to early parameters requires multiplying together many Jacobian matrices — one per time step. The formula is:

∂E₃/∂W = Σₖ₌₀³ (∂E₃/∂ŷ₃)(∂ŷ₃/∂s₃)(∂s₃/∂sₖ)(∂sₖ/∂W)

The term ∂s₃/∂sₖ involves a product of Jacobians: ∏ⱼ ∂sⱼ/∂sⱼ₋₁. Since tanh maps values to (-1,1) and its derivative is bounded by 1, and the Jacobian 2-norm has an upper bound of 1, this product shrinks exponentially with time. Gradient contributions from distant time steps become effectively zero — the model cannot learn long-range dependencies. Depending on activation and weights, gradients can also explode (>1 repeatedly multiplied).
Q2. What is the difference between short-term and long-term dependencies in RNN?
  • Short-term dependency: The relevant context is close to the prediction point. Example: "The clouds are in the ___" — just the last 2-3 words matter. Standard RNN handles this well because gradients don't need to travel far.
  • Long-term dependency: The relevant context is far back in the sequence. Example: "I grew up in India… I speak fluent ___" — "Hindi" depends on "India" which appeared many steps ago. Standard RNN fails here because gradients vanish before reaching early time steps.

LSTM/GRU solve long-term dependencies through their gating mechanisms and cell state (constant error carousel).

Q3. Explain the LSTM gates — what does each one do?

LSTM has 3 sigmoid gates (each outputs 0–1, acting as a percentage/valve) + 1 tanh candidate:

  • Forget Gate fₜ = σ(Wf·[hₜ₋₁, xₜ] + bf) — Decides what fraction of the old cell state Cₜ₋₁ to keep. fₜ≈0 = forget everything, fₜ≈1 = keep everything. Example: when reading a new subject in a sentence, forget the old subject's gender.
  • Input Gate iₜ = σ(Wi·[hₜ₋₁, xₜ] + bi) — Decides how much of the new candidate information to write to memory. Works together with C̃ₜ.
  • Candidate C̃ₜ = tanh(Wc·[hₜ₋₁, xₜ] + bc) — Creates candidate new values (range -1 to +1) using current input and previous hidden state. This is the "what to potentially add."
  • Output Gate oₜ = σ(Wo·[hₜ₋₁, xₜ] + bo) — Decides which parts of the cell state to output as hₜ. Applies tanh to cell state first, then gates it.

f and i are different because Wf ≠ Wi — they are separate learned weight matrices.

Q4. What is the Cell State in LSTM, and why is it called "long-term memory"?

The cell state Cₜ is a vector that runs horizontally through the LSTM chain with only minor linear interactions (forget gate multiplication + input gate addition). The update rule is:

Cₜ = fₜ ⊙ Cₜ₋₁ + iₜ ⊙ C̃ₜ

It is called long-term memory because: (1) it is not passed through tanh before being carried forward — so it doesn't get squeezed; (2) the gradient flows through it via ∂Cₜ/∂Cₜ₋₁ = fₜ, which the network learns to keep near 1 when memory must be preserved; (3) information stored in Cₜ at an early step can persist for many steps without vanishing.

The hidden state hₜ = oₜ ⊙ tanh(Cₜ) is the short-term memory — it is the actual output at each time step and changes more drastically.

Q5. What is the "Constant Error Carousel" in LSTM?

The Constant Error Carousel (CEC) refers to the self-recurrent connection in the memory cell where the weight is fixed at 1 (not learned, not decayed). This creates a direct path for gradient to flow backwards through time without multiplicative decay.

In mathematical terms: the gradient ∂Cₜ/∂Cₜ₋₁ = fₜ. If the forget gate learns to output values near 1 when memory should be preserved, this product ∏fⱼ ≈ 1 — the gradient doesn't vanish. The gates regulate whether information enters or exits, but the cell state itself maintains a "constant error" highway.

Q6. What are the differences between LSTM and GRU?
AspectLSTMGRU
States2 (Cₜ + hₜ)1 (hₜ only)

ParametersMore (4 weight matrices)Fewer (3 weight matrices)

GRU merges the forget and input gate into a single update gate zₜ. When zₜ is high, more new information is used; when zₜ is low, old hidden state is preserved. GRU has no separate cell state — everything is in hₜ.

Q7. Explain the GRU Update Gate and Reset Gate — what is the intuition?
  • Update Gate zₜ = σ(Wz·xₜ + Uz·hₜ₋₁): Controls how much of the past hidden state to carry forward. It is the interpolation coefficient in hₜ = (1−zₜ)⊙hₜ₋₁ + zₜ⊙h̃ₜ. If zₜ≈0: keep the old state almost entirely (memory preserved). If zₜ≈1: replace with new candidate. This is the combined effect of LSTM's forget AND input gates.
  • Reset Gate rₜ = σ(Wr·xₜ + Ur·hₜ₋₁): Controls how much of the previous hidden state to use when computing the new candidate h̃ₜ = tanh(Wh·xₜ + rₜ ⊙ Uh·hₜ₋₁). If rₜ≈0: ignore previous hidden state when creating new candidate (equivalent to starting fresh). If rₜ≈1: use full previous context. This gate allows the model to drop irrelevant history.
Q8. What is BPTT (Backpropagation Through Time)? Why are hidden states saved in memory?

BPTT is the algorithm for training RNNs. Since the same weights (U, W, V) are shared across all time steps, the total gradient is the sum of gradients from each time step: ∂E/∂W = Σₜ ∂Eₜ/∂W.

Computing ∂Eₜ/∂W requires going back through all previous time steps via chain rule. For example, ∂E₃/∂W requires ∂s₃/∂s₂, ∂s₂/∂s₁, ∂s₁/∂s₀ — all intermediate hidden states.

This is why all hidden states h₀, h₁, ..., hₜ must be stored in memory during the forward pass — they are needed during the backward pass to compute these chain rule terms. This creates a memory constraint because sequences can be very long (e.g., h₀ through h₁₀,₀₀₀ must all be stored).

Q9. Why can't we use BiRNN for language generation / real-time applications?

BiRNN requires the entire sequence to be available before processing begins — the backward RNN reads from right to left, so it needs all future tokens. This makes BiRNN unsuitable for:

  • Language generation (e.g., GPT): At each step, you must generate the next token without seeing future tokens. Using BiRNN would constitute cheating — the model would see the very word it's supposed to predict.
  • Real-time speech recognition: Future audio hasn't been recorded yet.
  • Online/streaming tasks: Data arrives sequentially and predictions must be made immediately.

BiRNN is appropriate for tasks where the entire input is available before producing output: sentiment analysis, NER, machine translation encoding, question answering.
