RNN/LSTM শেখার পরেও সমস্যা কোথায়?

তুমি LSTM দিয়ে sentiment classification বা text generation করতে পারো — এগুলো fixed-to-sequence বা sequence-to-fixed length কাজ।

🚨 আসল সমস্যা: Machine Translation-এ input sequence-এর length এবং output sequence-এর length সম্পূর্ণ আলাদা হতে পারে।

উদাহরণ: "Think about it" (৩টি word) → "ওটা ভাবো" (২টি word)
"Come in" (২টি word) → "ভিতরে এসো" (২টি word)

Plain RNN/LSTM এই variable-length to variable-length mapping করতে পারে না!

তোমার নোটেও লেখা আছে:

Attention কেন পড়তে হলো? (The Bottleneck Problem)

Encoder-Decoder architecture দিয়ে Seq2Seq সমস্যা solve হলো ঠিকই, কিন্তু একটা নতুন সমস্যা এলো:

🍾 Context Vector Bottleneck:
Encoder পুরো input sentence-এর সব information মাত্র একটা fixed-size vector (ht, ct) এ compress করে দেয়। Long sentence হলে এই ছোট্ট vector সব information ধরে রাখতে পারে না!

যেমন: ৫০ word-এর একটা sentence translate করতে হবে। Encoder শুধু শেষের hidden state পাঠালে প্রথমের word-এর information হারিয়ে যায়।
💡 Attention Solution:
Attention mechanism বলে — "শুধু শেষের hidden state না, সব timestep-এর hidden state decoder-এ পাঠাও। Decoder নিজেই dynamically decide করবে কোন timestep-এর দিকে কতটা focus করতে হবে।"
STEP 1 — RNN/LSTM
Sequence data process করতে শিখলাম। Fixed-length output।
STEP 2 — Seq2Seq (Encoder-Decoder)
Variable-length input → Variable-length output handle করতে শিখলাম।
STEP 3 — Problem!
Long sentence-এ Context Vector bottleneck। Information loss।
STEP 4 — Attention Mechanism
সব encoder hidden states ব্যবহার। Dynamic focus। Long sentence-এও accurate translation।
Vanilla (Plain) Encoder-Decoder Architecture — কোনো Attention ছাড়া। End-to-end complete flow।

ENCODER — বিস্তারিত

Encoder হলো একটি LSTM যেটি input sentence-কে token by token (word by word) process করে।

Encoder কী করে?
১. প্রতিটি word-কে one-hot বা embedding vector-এ convert করে।
২. LSTM-এর মাধ্যমে sequentially process করে।
৩. প্রতিটি timestep-এ একটা hidden state (ht) এবং cell state (ct) তৈরি করে।
৪. কিন্তু intermediate outputs ignore করা হয়।
৫. শুধু final ht এবং ct context vector হিসেবে decoder-কে দেওয়া হয়।
t=1 → "Think" input করো
LSTM: (h₀, c₀) + x₁ → (h₁, c₁). Output ignore।
t=2 → "about" input করো
LSTM: (h₁, c₁) + x₂ → (h₂, c₂). Output ignore।
t=3 → "it" input করো
LSTM: (h₂, c₂) + x₃ → (h₃, c₃). Output ignore।
Final → Context Vector!
(h₃, c₃) = Context Vector → Decoder-এ পাঠানো হয়।
Encoder final state: h_T, c_T (T = last timestep) Context Vector = (h_T, c_T) = পুরো input sequence-এর "memory"
🔓
DECODER — বিস্তারিত

Decoder-ও একটি LSTM। এটি Encoder-এর Context Vector (ht, ct) দিয়ে শুরু করে এবং target language-এ একটি একটি করে word generate করে।

Decoder কীভাবে কাজ করে?
১. প্রথম input: <START> token।
২. Initial hidden state: Encoder-এর h_T এবং c_T।
৩. প্রতিটি timestep-এ Softmax layer দিয়ে পরের word predict করে।
৪. যে word predict হয় সেটাই পরের timestep-এ input হয়।
৫. <END> token predict হলে থামে।
t=1 → <START> input
(h₃, c₃) + <START> → Softmax → "ওটা" predict
t=2 → "ওটা" input
(h_d1, c_d1) + "ওটা" → Softmax → "ভাবো" predict
t=3 → "ভাবো" input
(h_d2, c_d2) + "ভাবো" → Softmax → <END> predict → থামো!
🎯
Training Process — Loss ও Backpropagation
Teacher Forcing কী?
Training-এর সময় Decoder-এ previous timestep-এর prediction-এর বদলে actual correct word input হিসেবে দেওয়া হয়। এতে training faster হয় কিন্তু inference-এ actual prediction ব্যবহার হয়।

Loss Function: Categorical Cross Entropy (multiclass classification)

L = - Σ (i=1 to N) [ y_true_i × log(y_pred_i) ] উদাহরণ: y_true = [0, 1, 0, 0, 0, 0] (সঠিক word = "ওটা") y_pred = [0.1, 0.6, 0.1, 0.05, 0.1, 0.05] (softmax output) L = -1 × log(0.6) = -(-0.51) = 0.51

⬆️
Vanilla Architecture-এর ৩টি Improvement
1️⃣ Embeddings
One-hot vector-এর বদলে dense embedding ব্যবহার। Word2Vec-এর মতো pretrained embedding বা model-এর সাথে jointly train করা embedding। Lower dimension, richer meaning।
2️⃣ Deep LSTMs
Single layer LSTM-এর বদলে multiple stacked LSTM layers। Long-term dependency ভালো capture করে। Word → Sentence → Paragraph level hierarchy।
3️⃣ Reversing Input
Input sequence উল্টো করে দাও। "Think about it" → "it about Think"। প্রথম word-এর gradient propagate করা easier হয়। Short-term dependencies আরও ভালো কাজ করে।
⚠️ তবুও সমস্যা:
এই সব improvement-এর পরেও long sentence-এর জন্য Context Vector Bottleneck রয়েই যায়। তাই Attention দরকার।

📡
Attention-এর মূল ধারণা

Vanilla model-এ Encoder শুধু শেষের hidden state পাঠায়। কিন্তু Attention model-এ:

Encoder সব timestep-এর hidden state পাঠায়!
যদি input "turn off the lights" (৪টি word) হয়, তাহলে h₁, h₂, h₃, h₄ — সব পাঠায়।

Decoder প্রতিটি timestep-এ নিজেই calculate করে কোন encoder hidden state-গুলো কতটা গুরুত্বপূর্ণ।

তোমার নোটের উদাহরণ: "turn off the lights" → হিন্দিতে "লাইট বন্ধ করো"

  • Decoder যখন "লাইট" generate করছে → encoder-এর "lights" timestep-এর দিকে বেশি attention
  • Decoder যখন "বন্ধ" generate করছে → encoder-এর "off" timestep-এর দিকে বেশি attention
🧮
Attention Weight (α) Calculation — Step by Step
Notation:
• h_j = Encoder-এর j-th timestep hidden state
• s_{i-1} = Decoder-এর (i-1)-th timestep hidden state (previous)
• α_{ij} = Decoder-এর i-th timestep যখন Encoder-এর j-th word-এর দিকে কতটা focus করবে
• C_i = Decoder-এর i-th timestep-এর জন্য context vector
STEP 1 — Score Calculate

Alignment score calculate করো: e_{ij} = f(h_j, s_{i-1})

f হলো একটা small ANN (Artificial Neural Network) — যা universal function approximator। এটা encoder এবং decoder hidden state নিয়ে একটা scalar score দেয়।

e_{ij} = ANN(h_j, s_{i-1}) → একটা number (score)
STEP 2 — Softmax দিয়ে Normalize

সব score-গুলোকে softmax দিয়ে probability-তে convert করো (sum = 1):

α_{ij} = softmax(e_{ij}) = exp(e_{ij}) / Σ_k exp(e_{ik}) সব α_{ij} এর sum = 1 (i fixed রেখে)
STEP 3 — Weighted Sum = Context Vector

Attention weights দিয়ে encoder hidden states-এর weighted sum নাও:

C_i = Σ_j (α_{ij} × h_j) = α_{i1}×h₁ + α_{i2}×h₂ + α_{i3}×h₃ + α_{i4}×h₄

C_i একটি vector (h-এর মতো same dimension)।

STEP 4 — Decoder-এ Use করো
Decoder input at step i = [y_{i-1}, s_{i-1}, C_i] তারপর Softmax → word predict

Step 4: Loss Calculation (Categorical Cross Entropy)

তোমার নোটের উদাহরণ অনুযায়ী loss calculation:

Loss Formula: L = -Σ (y_true_i × log(y_pred_i)) ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ Timestep t=1: y_true = [0, 1, 0, 0, 0, 0, 0] (correct = "ওটা", index 1) y_pred = [0.13, 0.26, 0.08, 0.07, 0.06, 0.06, 0.06] (softmax) L₁ = -(0×log(0.13) + 1×log(0.26) + 0×... ) = -1 × log(0.26) = -1 × (-1.347) = 1.347 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ Timestep t=2: y_true = [0, 0, 1, 0, 0, 0, 0] (correct = "ভাবো", index 2) y_pred = [0.04, 0.08, 0.45, 0.12, 0.11, 0.10, 0.10] L₂ = -1 × log(0.45) = -1 × (-0.799) = 0.799 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ Timestep t=3: y_true = [0, 0, 0, 0, 0, 0, 1] (correct = <END>, index 6) y_pred = [0.03, 0.04, 0.05, 0.06, 0.04, 0.05, 0.73] L₃ = -1 × log(0.73) = -1 × (-0.315) = 0.315 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ Total Loss = L₁ + L₂ + L₃ = 1.347 + 0.799 + 0.315 = 2.461 (Training continue হলে loss কমতে থাকে)
নোটের example-এর সাথে মিল:
তোমার নোটে দেখানো হয়েছিল: L_{t=1} = -1×log(0.1) = 1, L_{t=2} = 1, L_{t=3} = -1×log(0.4) = 0.39। এটাই same calculation!

🟢 Conceptual / Foundational Questions
Q-01
Why can't a standard RNN or LSTM directly perform machine translation? What is the fundamental limitation?
Hint: Variable-length input and output sequences. Fixed hidden state size cannot capture all information.
conceptual
Q-02
What is the role of the Context Vector in the Encoder-Decoder architecture? What information does it carry?
Hint: (h_T, c_T) — compressed representation of the entire input sequence.
conceptual
Q-03
Why are the outputs of the Encoder LSTM ignored during encoding? What is the significance of only using the final hidden state?
Hint: Each intermediate hidden state is a partial representation. Only the final state has seen all input tokens.
conceptual
Q-04
Explain the role of <START> and <END> tokens in the Decoder. Why are they necessary?
Hint: <START> signals beginning of generation. <END> tells the model when to stop.
conceptual
Q-05
What is Teacher Forcing? What are its advantages and disadvantages?
Hint: Feeding true labels during training vs predicted output. Pros: faster convergence. Cons: exposure bias at inference.
conceptual
🟡 Architecture-specific Questions
Q-06
What is the difference between the input to the Decoder at each timestep in a Vanilla Encoder-Decoder vs an Attention-based Encoder-Decoder?
Hint: Vanilla = [y_{i-1}, s_{i-1}]. Attention = [y_{i-1}, s_{i-1}, C_i]. C_i is the dynamic context vector.
architecture
Q-07
Explain the "Context Vector Bottleneck" problem. How does Attention mechanism solve it?
Hint: Fixed-size vector must compress entire sentence. Attention uses ALL encoder hidden states + dynamic weighting per decoder step.
attention
Q-08
What is α_{ij} in the Attention mechanism? How is it computed and what does it represent?
Hint: Alignment/similarity score between decoder timestep i and encoder timestep j. Computed via ANN(h_j, s_{i-1}) → softmax.
attention
Q-09
Why does the attention weight matrix (α) sum to 1 across each row? What mathematical property ensures this?
Hint: Softmax function applied to scores. Output sums to 1, making it a valid probability distribution over source positions.
attention
Q-10
How is the dynamic Context Vector C_i computed in the Attention mechanism? Write the formula.
Hint: C_i = Σ_j (α_{ij} × h_j) — weighted sum of ALL encoder hidden states.
attention
🔴 Advanced / Tricky Questions
Q-11
In Attention, we say α_{ij} = f(h_j, s_{i-1}). Why is s_{i-1} used instead of s_i? Why the "previous" decoder hidden state?
Hint: At decoding step i, s_i hasn't been computed yet — it depends on C_i which we're trying to calculate. Causality constraint.
tricky
Q-12
Why is the Alignment score function f in Attention implemented as a small ANN (neural network) rather than a simple dot product?
Hint: ANN = Universal Function Approximator. Can learn complex non-linear relationships between h_j and s_{i-1}. Dot product is too simplistic (Bahdanau vs Luong attention).
advanced
Q-13
What are the three improvements to the vanilla Seq2Seq model (as per your notes)? Explain the intuition behind "Reversing the Input."
Hint: 1) Embeddings, 2) Deep LSTMs, 3) Reversing input. Reversing: first source word becomes closest to first target word → shorter gradient path.
improvements
Q-14
During Attention training, how does the ANN that computes alignment scores get trained? Is it trained separately?
Hint: No — it is trained jointly (end-to-end) with the entire Encoder-Decoder network via backpropagation.
training
Q-15
Why is loss calculated using Categorical Cross Entropy and not Mean Squared Error for machine translation?
Hint: Translation is multiclass classification (predict next word from vocabulary). CCE is designed for probability distributions; MSE doesn't work well with softmax outputs.
loss
Q-16
In the Attention mechanism, what is a "Time Distributed Dense Layer" and why is it needed?
Hint: Applies the same Dense/Linear layer to each timestep independently. Used to compute alignment scores e_{ij} = W·h_j across all encoder timesteps in parallel.
architecture
Q-17
Suppose you have a 50-word input sentence. What is the size of the attention weight matrix (α)? How many context vectors are computed if the output is 30 words?
Hint: Matrix size = 30 (decoder steps) × 50 (encoder steps). Context vectors = 30 (one per decoder timestep).
calculation
Q-18
What is Deep LSTM in the context of Seq2Seq? What does "Layered Representation" mean (word → sentence → paragraph)?
Hint: Multiple LSTM layers stacked. Lower layers capture lexical/word-level patterns; higher layers capture semantic/discourse-level patterns.
