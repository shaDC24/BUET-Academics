PART 3 Initial Weight Matrices (Random Initialization)
In SGNS there are TWO separate matrices:
  • W (input/embedding matrix, V×N = 5×3) — stores the embedding of each word when it acts as a CENTER word.
  • W' (context/output matrix, V×N = 5×3) — stores a separate vector for each word when it acts as a CONTEXT word.
Both are initialized randomly. After training, W is used as the final word embeddings.

1-C
Apply Sigmoid σ(x) = 1/(1 + e^(−x))
Positive pair: σ(s_pos) → want this near 1 (they ARE neighbors)
Negative pair: σ(−s_neg) → want this near 1 (this = probability they are NOT neighbors)
━━━ Positive ━━━
σ(s_pos)  = σ(0.740) = 1/(1 + e^(−0.740)) = 1/(1 + 0.4772) = 0.677
Interpretation: model says 67.7% chance campusx & for are neighbors. Want → 100%

━━━ Negative 1 ━━━
σ(−s_neg1) = σ(−0.310) = 1/(1 + e^(0.310)) = 1/(1 + 1.3634) = 0.423
Interpretation: model says 42.3% chance they are NOT neighbors. Want → 100%

━━━ Negative 2 ━━━
σ(−s_neg2) = σ(−0.290) = 1/(1 + e^(0.290)) = 1/(1 + 1.3365) = 0.428
Interpretation: model says 42.8% chance they are NOT neighbors. Want → 100%
1-D
Compute Loss L = −log σ(s_pos) − log σ(−s_neg1) − log σ(−s_neg2)

1-E
Compute Gradients (Chain Rule)
Key formulas:
 ∂L/∂s_pos   = −(1 − σ(s_pos))  ← error for positive pair
 ∂L/∂s_neg_k = σ(s_neg_k)          ← error for each negative pair
 Then: ∂L/∂v_center = Σ (∂L/∂s_k) × v'_context_k   (sum over pos + all negs)
 And:  ∂L/∂v'_context_k = (∂L/∂s_k) × v_center
━━━ Step 1: Scalar error terms ━━━
err_pos = −(1 − σ(0.740)) = −(1 − 0.677) = −0.323
err_neg1 = σ(s_neg1)       = σ(0.310)      = 0.577
err_neg2 = σ(s_neg2)       = σ(0.290)      = 0.572

# σ(0.310) = 1/(1+e^−0.310) = 1/(1+0.7334) = 0.577
# σ(0.290) = 1/(1+e^−0.290) = 1/(1+0.7483) = 0.572

━━━ Step 2: Gradient w.r.t. v_campusx (center word in W) ━━━

Key Insights from the simulation
1. Two separate matrices: W and W' serve different roles. W gives input embeddings (the final word vectors we care about). W' gives context embeddings (used only during training, discarded after).

2. Positive pair = attraction: v'_for got pulled toward v_campusx because they kept appearing together. The gradient was negative (−0.129, −0.162, −0.194) on W'[2], which reduced the error between them.

3. Negative pair = repulsion: v'_data and v'_science got pushed away from campusx and for. Their vectors shrank in the dimensions where campusx is strong.

4. Only updated words: In each step, only the center word's row in W and the positive + negative words' rows in W' are updated. Other rows are untouched — this is why SGNS is so fast.

5. No Softmax needed: Instead of computing probabilities over all V=5 words (or 3 million in real use), we only compute 3 sigmoid values per step (1 positive + 2 negatives). Speed-up is enormous.
